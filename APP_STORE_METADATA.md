> All screenshots must be PNG or JPEG, RGB color space, no alpha channel.

- Overlay captions in a design tool — plain screenshots without captions historically underperform
- Avoid showing any placeholder/mock data; use realistic task titles, real-looking names, and dollar amounts

## 11. Review Notes for App Store Reviewer

**Copy this into the "Notes" field on the App Store Connect submission page.**

```
Hello App Review Team,

Thank you for reviewing HustleXP. Here is key context to help with your review:

WHAT THE APP DOES:
HustleXP is a two-sided gig economy marketplace. "Posters" post tasks they need
completed (home services, deliveries, errands, skilled work). "Hustlers" browse
tasks, apply, complete them, and earn money + XP points. All financial
transactions are processed via Stripe and held in escrow until work is confirmed.

USER ROLES:
The app has two primary user types selectable during onboarding:
1. Hustler — browses and applies for tasks, earns money
2. Poster — posts tasks, reviews applicants, manages work

TEST ACCOUNT CREDENTIALS:
Please use the following sandbox test accounts to review both flows:

Hustler account:
  Email: testreviewer+hustler@hustlexp.app
  Password: HXReview2026!

Poster account:
  Email: testreviewer+poster@hustlexp.app
  Password: HXReview2026!

NOTE: These accounts exist in our staging environment. If you encounter any
login issues, please contact us at support@hustlexp.app and we will assist
within 24 hours.

PAYMENT TESTING:
All payments use Stripe in test/sandbox mode. No real charges will occur.
Use Stripe test card 4242 4242 4242 4242 (any future expiry, any CVC) if
a payment flow is triggered during review.

SUBSCRIPTION IN-APP PURCHASES:
Two auto-renewable subscription products are included (Premium $9.99/mo,
Pro $19.99/mo). These can be tested using the standard App Store sandbox
environment. The paywall is accessible from the Profile tab > "Upgrade" button.

KYC / IDENTITY VERIFICATION:
The app requires identity verification (KYC) before a user can transact.
For the reviewer accounts above, KYC has been pre-completed so you can
access all features immediately without needing to submit personal documents.

USER-GENERATED CONTENT:
The app includes user-generated content (task descriptions, chat messages,
photos). We have a content moderation policy and reporting mechanism in place.
Users can report inappropriate content from any task or message screen via
the "..." menu → "Report."

LIVE BROADCASTS:
The "Live Broadcasts" feature allows Posters to broadcast urgent tasks in
real time. This is standard application networking (WebSockets/SSE), not
a video/audio broadcast. No camera or microphone is used for this feature.

LOCATION PERMISSION:
The app requests "When In Use" location access to show nearby tasks.
Location is not used for background tracking.

PHOTO LIBRARY PERMISSION:
Used only when a Hustler submits proof-of-work photos or when a user
sends a photo in chat. The app never uploads photos without explicit
user action.

DISPUTE RESOLUTION:
The app includes a dispute resolution system where users can flag a
completed task as disputed. This involves submitting text and photo
evidence. An internal moderation team (not AI-only) reviews disputes.
This is referenced in our terms of service.

PUSH NOTIFICATIONS:
The app requests notification permission for task updates, chat messages,
and broadcast alerts. Notifications are not sent without user permission.

If you have any questions or need additional test accounts for specific
scenarios, please do not hesitate to reach out.

Thank you,
The HustleXP Team
support@hustlexp.app
```
