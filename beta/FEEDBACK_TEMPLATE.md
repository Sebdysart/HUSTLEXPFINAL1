# Beta Feedback Template

Use this template when reporting bugs, requesting features, or asking questions during the HustleXP private beta.

**Submit to**: beta@hustlexp.app
**Subject line format**: `[BUG] Short description` / `[FEATURE] Short description` / `[QUESTION] Short description`

You can also shake your device in the app to auto-populate a report with a screenshot attached.

---

## Report Type

Select one:

- [ ] Bug — something is broken or behaves incorrectly
- [ ] Feature Request — something you wish the app did
- [ ] Question — you're unsure how something works
- [ ] Performance Issue — slowness, crashes, battery drain
- [ ] UI / UX Feedback — design, layout, or flow feels off

---

## Summary

*One sentence describing the issue or request.*

> Example: "The Apply button on the task detail screen does nothing when tapped."

---

*For bugs: exact steps from a fresh app state that reproduce the issue. Be specific.*

4.

> 2. Tap the Feed tab.

## Expected Behavior

*What should have happened?*

> Example: "A text field should appear asking me to write my application pitch."

---

## Actual Behavior

*What actually happened instead?*

> Example: "The button appears to do nothing. No error message, no navigation, no visual feedback."

---

## Severity

Select one:

- [ ] **Blocker** — cannot proceed, core flow is broken (e.g., can't submit a task, can't release payment)
- [ ] **Major** — significant issue that degrades the experience but has a workaround
- [ ] **Minor** — noticeable issue with minimal impact on core functionality
- [ ] **Cosmetic** — visual-only issue (alignment, truncation, wrong color, etc.)
