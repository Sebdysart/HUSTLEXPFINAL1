# Archetype C: Task Lifecycle Screens

**Status:** 🔄 Transform after Feeds

## Screens
- TaskDetailScreen ✅
- TaskInProgressScreen
- HustlerEnRouteMapScreen
- HustlerOnWayScreen (poster side)
- TaskCompletionHustlerScreen
- TaskCompletionPosterScreen
- TaskReviewScreen

## Emotional Contract
> "This is already in motion."

## Visual Intent
- Simpler than feeds (simplicity = confidence)
- Clear progress indication
- Map integration feels natural
- Status updates feel automatic
- Completion feels inevitable

## Allowed Atoms
- HScreen (ambient=true)
- HCard (status cards, info sections)
- HMoney (payment display)
- HText (clear hierarchy)
- HBadge (status: "En route", "In progress")
- HButton (action CTAs)
- HActivityIndicator (live status)
- HSignal (status updates)

## Forbidden
- ❌ Complex UI
- ❌ Multiple CTAs competing
- ❌ Anxiety-inducing elements
- ❌ Countdown timers (unless truly necessary)
- ❌ "Waiting for..." language

## Status Language
Instead of "Waiting for hustler" → "On the way"
Instead of "Task pending" → "In motion"
Instead of "Payment processing" → "Payment secured"

## CTA Language
- "I'll do this" (accept task)
- "I'm here" (arrived)
- "Done" (completion)
- "Looks good" (approval)

## Map Screens
- Map is background, not focus
- Status card overlays map
- Minimal controls
- ETA shown simply

## Completion Screens
- Celebratory but not over-the-top
- Money earned prominent
- XP gained shown
- "Another one done" energy
- Quick path to next task
