# Archetype F: System / Interrupt Screens

**Status:** 🔄 Transform last

## Screens
- ChatScreen
- TaskConversationScreen
- NotificationsScreen
- DisputeEntryScreen
- EligibilityMismatchScreen
- InstantInterruptCard
- SettingsScreen
- WalletScreen

## Emotional Contract
> "The system has this under control."

## Visual Intent
- Calm, neutral, resolved
- Authority without aggression
- Clear information hierarchy
- Actions obvious but not urgent
- Trust-preserving

## Allowed Atoms
- HScreen (ambient=false for some)
- HCard (content containers)
- HText (clear, neutral)
- HBadge (status indicators)
- HButton (actions)
- HInput (for disputes, chat)

## Forbidden
- ❌ Neon accents
- ❌ Urgent language
- ❌ Alarm colors (unless true error)
- ❌ Exclamation points
- ❌ "Warning!" patterns

## Chat Screens
- Messages feel natural
- Bubbles: theirs = dark card, yours = purple subtle
- Input at bottom, always visible
- Send button obvious
- No typing indicators (feels watched)

## Notification Screens
- Grouped by type/time
- Read/unread subtle
- Tap to navigate to source
- Clear all option available

## Dispute Screens
- Calm, factual tone
- Step by step guidance
- "We'll help sort this out"
- Never feels adversarial

## Settings Screens
- Grouped logically
- Toggles for booleans
- Navigation for sub-screens
- No overwhelming options

## Wallet Screens
- Balance prominent
- Transaction history clean
- Add/withdraw obvious
- Security feels solid

## Copy Rules
- "We've got this" not "Action required"
- "Let us know" not "Report issue"
- "Your balance" not "Wallet balance"
- Neutral, professional, trustworthy

## Error States
- Never blame user
- Explain simply
- Offer clear next step
- "Hmm, that didn't work" energy
